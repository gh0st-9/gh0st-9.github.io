浅谈Java下的命令执行

常见利用方式

Runtime.exec()

java.lang.ProcessBuilder

java.lang.ProcessImpl

javax.script.ScriptEngineManager

Runtime.exec()

exec()方法带有多种重载方法，envp为环境，dir为目录

传入形式为String和String[]

跟进传入String的exec()进行调试

接收String command后调用public Process exec(String command, String[] envp, File dir)

随后String command经过StringTokenizer类实例后变成String[] cmdarray

跟进StringTokenizer

添加空格、tab、回车换行、换行、换页符为分隔符，将传入的 command 分割，获得最终的String[]，发现本质还是要转换为String[]来进行命令的调用，最后调用public Process exec(String[] cmdarray, String[] envp, File dir)执行命令

public Process exec(String[] cmdarray, String[] envp, File dir)

通过实例化 ProcessBuilder 来处理传入的 cmdarray。说明Runtime.getRuntime.exec()的底层实际上是ProcessBuilder。到这里最外层的exec的调用就结束了，下面就是ProcessBuilder的调用了

ProcessBuilder.start()

可以直接用ProcessBuilder.start()来执行命令

java.lang.ProcessBuilder#start

这里将cmdarray[0]赋值给prog，随后判断security是否为null，如果SecurityManager开启,会调用SecurityManager#checkExec()对执行程序prog进行检查，然后调用java.lang.ProcessImpl#start

ProcessImpl.start()

ProcessImpl为更下面一层，Runtime和ProcessBuilder执行命令实际上最后是调用ProcessImpl这个类，值得注意的是，需要使用反射的方式来调用ProcessImpl类从而进行命令执行

java.lang.ProcessImpl#start

下面分两种情况，Windows 会调用 ProcessImpl 类的构造方法，Linux 会调用 java.lang.UNIXProcess#init<> 。
