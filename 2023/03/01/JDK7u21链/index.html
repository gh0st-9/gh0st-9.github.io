Java安全之路--JDK7u21链

前言

7u21链在不需要引进第三方库的情况下, 可以利用jdk的原生代码实现反序列化漏洞

jdk版本<=7u21

一、环境搭建

1、jdk-7u21

2、需要导入javassist，以用于为了方便生成恶意类的字节码

二、反序列化Gadgets

yso链

核心利用点在AnnotationInvocationHandler类

在他的invoke方法中，如果传入的方法为equals且只有一个Object类型形参的时候，就可以进入equalsImpl()方法。

跟进equalsImpl

存在var8 = var5.invoke(var1);语句, 可以通过反射调用var1对象的var5方法，所以var1可以设置为TemplatesImpl类，var5为TemplatesImpl类的newTransformer或getTransletInstance方法，这样就可以执行类加载恶意字节码，从而执行恶意代码

我们可以通过AnnotationInvocationHandler创建任意一个代理对象，然后代理对象调用equals方法传入参数为恶意TemplatesImpl对象，来进行测试

demo1

这里讲下var2的来历

跟进getMemberMethods

这里通过反射获得this.type中的所有方法。而this.type是在该对象构造方法中传入的第一个参数。

但是想要走到var5.invoke(var1), 需要满足3个if判断

  • var1 != this;
  • var1对象需能够转化为 this.type 类型，this.type 应该为 var1 对应类的本类或父类;
  • this.asOneOfUs(var1) 返回 null.z

接下来就是去找哪里调用了equals

在LinkedHashSet类，它继承了HashSet类，同时它的readObject方法也是从HashSet继承的

重点是在他的for循环：会将传入的序列化对象进行遍历，待其反序列化后，调用put

跟进put

看到if (e.hash == hash && ((k = e.key) == key || key.equals(k)))，其中key和k都可控

控制key为恶意TemplatesImpl，k为恶意AnnotationInvocationHandler代理对象。

但是想要走到key.equals(k), 需要满足2个if判断

  • e.hash == hash为true
  • (k = e.key) == key为false

看到 int hash = hash(key)，跟进hash

最后会执行k.hashCode() ，而k为所传入的参数。

所以令k为恶意AnnotationInvocationHandler代理对象，就会调用代理对象中的invoke方法。那么实际上最注重会执行到AnnotationInvocationHandler的hashCodeImpl方法

跟进hashCodeImpl

该方法的返回值是一个 int 类型的哈希值，表示该对象在散列表中的位置。

在方法中，首先将变量 var1 初始化为0。然后使用一个 for 循环遍历成员变量的 Map 对象 memberValues 中的所有键值对。在循环中，使用变量 var3 来保存当前键值对，然后将其键的哈希值乘以127，并使用 memberValueHashCode() 方法计算其值的哈希值。memberValueHashCode() 方法用于计算该成员变量值的哈希码。最后将这个值累加到 var1 中。

简单来说就是这个方法会遍历this.memberValues属性，然后对其中每一项键值属性进行进行位运算并累加

跟进memberValueHashCode

该方法首先获取对象的类型 var1，然后使用 if-else 语句判断该类型是否是数组类型。如果不是数组类型，则直接调用对象的 hashCode() 方法计算其哈希值并返回。

让memberValues这个Map只有一个键值对，让key的hash为0，127*0=0）。再让value是恶意的TemplatesImpl对象，这样计算的hash值就是 TemplatesImpl对象的hash值，所以就相同了。而hash为0的字符串，网上脚本跑出来的是f5a5a608
