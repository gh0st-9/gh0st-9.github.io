[NCTF 2022]calc

源码

非预期

可执行的地方有两处，eval和os.system，但是eval在这里过滤了太多函数而导致不可用，所以只能对os.system进行应用。

这里需要注意一个问题，如果直接传入命令，eval解析不了传入的东西是会直接报错的，从而不会进入到os.system，所以这里需要先绕过eval报错

利用python多行注释三引号'''绕过eval的报错，用 %0a 分隔不同命令, %09 代替空格

最终执行命令的地方是在log处，用echo来执行命令

思路是先用cp将flag内容带进一个文件，再利用curl将其带出来

payload

预期解

出题人发现python3的system具体实现后发现最后调用的位置，也是通过/bin/sh -c去执行的命令

根据p神的我是如何利用环境变量注入执行任意命令，核心思想就是通过设置环境变量来造成rce

而在python3中，环境变量是作为字典储存的，可以通过覆盖或者赋值来控制，参考Python黑魔法–绕过空格实现变量覆盖

所以通过这种方式去覆盖环境变量os.environ，设置os.environ['BASH_FUNC_echo%%']='() { id; }'，这样当调用echo函数的时候就会触发后面的命令，直接反弹shell

简单思路

首先绕过self.checkSign(),然后同时进入这两个判断if "scan" in self.action:,if "read" in self.action:，利用”scan”把flag写入到reasult.txt里，用”read”读取result.txt并放到result[‘data’]里，最后在/De1ta里用return json.dumps(task.Exec()) 以json形式返回出来

先看 if (self.checkSign()):，把相关代码都截取出来

self.checkSign()=>hashlib.md5(secert_key + param + action).hexdigest() == self.sign=>hashlib.md5(secert_key + 'flag.txt' + 'readscan').hexdigest() == self.sign

也就是需要secert_key + 'flag.txt' + 'readscan'的hash值

为什么这里action参数要控制为readscan？在进行if判断时，用的是in而不是==，而需求是要调用read和scan

虽然secert_key不知道，但是可以直接利用geneSign获取到secert_key + 'flag.txt' + 'readscan'的hash值

注意，在geneSign里已经给action进行了赋值

[NSSRound#7 Team]ShadowFlag

app.py

当接收到POST请求的/shell路由时，会执行shell()函数。该函数首先通过全局变量tag的值来判断是否为首次访问该路由，如果是，则将全局变量flag1删除并将tag标记为True，之后不再执行此判断。

接着，函数会删除系统中的flag1和flag2。

最后，函数会从POST请求的参数中获取名为”act”的值，如果不包含空格，则会执行命令

两个问题

  • 不能包含空格
  • 执行命令没有回显

反弹shell

空格可以用$IFS代替，没有回显那么就反弹shell

因为该题目环境是flask，所以可以利用python反弹shell，参考[PayloadsAllTheThings/Reverse Shell](https://github.com/swisskyrepo/PayloadsAllTheThings/blob/master/Methodology and Resources/Reverse Shell Cheatsheet.md#awk)

payload

前面说到，flag1是在/tmp目录下，但是在我们第一次访问/shell路由时就会执行os.system("rm -f /tmp/flag1.txt /tmp/flag2.txt")，将flag文件删除，但是注意flag1 = open("/tmp/flag1.txt", "r")，这里使用了 open 函数打开 flag1 文件，但没有使用 with 语句关闭文件句柄，因为flask进程还在运行，所以我们可以在/proc/[pid]/fd的内存中找到它

再看flag2.txt的操作过程

用with会默认打开使用后关闭这个文件，所以我们无法在进程中找到flag2，只有从环境里面去找，在高版本flask中，存在Flask debug PIN，访问 http://ip:port/console 路径输入正确PIN值即可进入调试环境

接下来就是计算pin值，这里是巨魔师傅计算pin值的脚本，其中有注释的地方都需要去计算

  • 第四个是在报错页面得到，在/shell路由传入不为act的POST数据

再看
