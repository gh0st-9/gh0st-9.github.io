Index

Flask

Index里就进行了一个session的赋值，其中key还是通过环境变量获取的

Admin路由需要session中name为admin，校验完成后使用pongo2进行模板渲染

Flask路由是一个开启debug的flask服务，然后返回flask服务页面的内容

首先伪造admin session，测试发现SESSION_KEY为空，所以本地直接起个服务伪造就行

main.go

下面看pango2渲染，并不能ssti，翻文档，看到一句话，pango2的模板规则和Django一样

配合gin框架的c.对象调用的方式，可构造去任意文件读取，记得url编码

c.Request.Header.Cmd|first是pango2中特有的用法，Pongo2中不直持访问结构体字段的语法，使用 c.Request.Header.Cmd是无法直接获取到Cmd的值。必须使用first过滤器获取请求头的值。

当时就去算pin码了，最后pin码成功算出来了，但没有Cookie执行不了命令

看到还剩一个路由/flask，因为开了debug，可报错得到其源码

reading

/books路由任意文件读取，rerplace掉了../，双写一下就行了

读取server.py

有/readflag，所以直接读/flag不可取

看代码

需要获取到secret_key和key这两个md5的值，然后用flasksession去伪造

解题：

  1. 读/proc/self/maps获取内存地址
  2. 从/proc/self/mem读取指定偏移的内存数据
  3. 找secret_key和key
  4. 伪造session

mem.py

完成后用正则匹配去找两个所需的md5值

然后在启动靶机时多打印几次当前行时间戳，以减少碰撞范围
